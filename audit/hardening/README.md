# Linux System Hardening and Security Audit

## Project Overview
This project demonstrates basic Linux security auditing and hardening techniques used to reduce system attack surface.

## Objectives
- Audit Linux system security
- Identify weak configurations
- Apply basic hardening steps
- Improve system security posture

## Tools Used
- Linux
- UFW Firewall
- SSH configuration

## Security Audit Performed
- User account review
- Service enumeration
- Open port analysis
- File permission checks

## Hardening Actions
- Disabled unused services
- Configured firewall rules
- Secured SSH access

## Outcome
- Reduced exposed services
- Improved system security
- Better understanding of Linux hardening

## Legal Disclaimer
All actions were performed on authorized systems for educational purposes only.
